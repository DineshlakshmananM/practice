C Program Structure

Basic Structure of a C Program

Every C program follows a specific structure. Understanding this structure is fundamental to writing C programs correctly. A typical C program consists of several sections.

Components of a C Program

  • Preprocessor Directives: Start with # symbol, processed before compilation
  • Comments: Explanation of code, ignored by compiler
  • Global Variables: Declared outside functions, accessible everywhere
  • Function Declarations: Function prototypes or signatures
  • Main Function: Entry point where program execution begins
  • Function Definitions: Implementation of user-defined functions

Program Structure Example

#include <stdio.h>           // Preprocessor directive
#include <stdlib.h>

// Single line comment
/* Multi-line comment */

int globalVar = 10;             // Global variable

// Function declaration (Prototype)
int add(int a, int b);
void displayMessage();

int main() {                    // Main function - entry point
    int a = 5, b = 3;           // Local variables
    int sum = add(a, b);        // Function call
    
    printf("Sum: %d\n", sum);
    displayMessage();
    
    return 0;                   // Return statement
}

// Function definitions
int add(int a, int b) {
    return a + b;
}

void displayMessage() {
    printf("Welcome to C!\n");
}

Execution Flow

1. Preprocessor processes directives (#include, #define)
2. Compilation translates code to machine code
3. Linking combines object files and libraries
4. Program starts executing from main() function
5. Functions are called as needed
6. Program returns control to OS when main() completes

Key Points

  • Every C program must have a main() function
  • main() is where execution begins and ends
  • Return 0 indicates successful execution
  • Curly braces {} define code blocks
  • Semicolons; terminate statements (except for loops and conditionals)
  • Comments help explain code and are ignored by compiler
